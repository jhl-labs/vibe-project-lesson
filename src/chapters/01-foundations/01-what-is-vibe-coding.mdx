# 바이브 코딩(Vibe Coding)이란?

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists." — Andrej Karpathy, 2025년 2월

## 개념의 탄생

**바이브 코딩(Vibe Coding)**은 2025년 2월, 전 Tesla AI 디렉터이자 OpenAI 공동창업자인 **Andrej Karpathy**가 X(트위터)에서 처음 사용한 용어입니다.

<Callout type="info" title="정의">
  바이브 코딩이란 개발자가 AI와 자연어 대화를 통해 코드를 생성하고, 코드의 세부 구현보다 
  의도(vibe)와 방향성에 집중하는 프로그래밍 방식입니다.
</Callout>

### 핵심 특징

- **자연어 중심**: 코드가 아닌 의도와 요구사항을 설명
- **AI 주도 구현**: AI가 실제 코드를 작성하고 개발자가 검증
- **반복적 대화**: 수정/개선을 대화를 통해 진행
- **코드 이해 선택적**: 모든 코드를 라인 단위로 이해할 필요 없음

## 장점과 한계

### 장점

- **생산성 향상**: 보일러플레이트, 반복 작업의 획기적 단축
- **진입장벽 낮춤**: 비전공자도 소프트웨어 개발 가능
- **프로토타이핑 가속**: 아이디어 → 동작하는 코드까지 빠른 전환
- **학습 도구**: 새 언어/프레임워크 학습 시 실시간 가이드

### 한계와 주의점

<Callout type="warning" title="주의사항">
  바이브 코딩의 편리함에만 의존하면 다음과 같은 위험이 있습니다.
</Callout>

- **코드 이해도 저하**: AI가 만든 코드를 이해하지 못하면 유지보수 불가
- **보안 취약점**: AI가 생성한 코드에 보안 허점이 포함될 수 있음
- **기술 부채 축적**: 검증 없이 빠르게 쌓인 코드는 장기적으로 문제
- **디버깅 어려움**: 이해하지 못한 코드의 버그 수정은 매우 어려움

## 현재의 위상

2025년 11월, Collins 사전은 **"Vibe Coding"**을 **올해의 단어(Word of the Year)**로 선정했습니다. 이는 AI 코딩이 더 이상 기술 분야만의 관심사가 아니라 사회 전반의 트렌드가 되었음을 보여줍니다.
