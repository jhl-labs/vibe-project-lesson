# Claude Code 플랫폼

> 터미널부터 브라우저, 데스크톱 앱, IDE, CI/CD, Slack까지 — 어디서든 실행되는 Claude Code

## 개요

Claude Code는 단일 터미널 CLI 도구로 시작했지만, 이제는 **9개 이상의 플랫폼**에서 실행됩니다. 개발자가 어디에 있든, 어떤 환경에서 작업하든, Claude Code의 에이전트 기능을 동일하게 사용할 수 있습니다. 터미널에서 시작한 세션을 웹으로 옮기고, Slack에서 동료에게 작업을 공유하며, GitHub Actions에서 자동화된 워크플로우를 실행하는 것이 모두 가능합니다.

<Callout type="info" title="핵심 철학">
  Claude Code의 플랫폼 전략은 **"개발자가 있는 곳에서 만나기"**입니다.
  모든 플랫폼에서 동일한 에이전트 아키텍처(Tool Loop, CLAUDE.md, MCP)가 작동하며,
  세션을 플랫폼 간에 자유롭게 이동할 수 있습니다.
</Callout>

## 1. Terminal (CLI) — 핵심 경험

터미널 CLI는 Claude Code의 **기본이자 핵심** 인터페이스입니다. 모든 플랫폼은 결국 이 CLI의 에이전트 아키텍처 위에서 동작합니다.

### 왜 터미널인가

개발자의 일상적인 작업 흐름 — 코드 편집, 빌드, 테스트, Git 커밋 — 은 터미널에서 시작하고 터미널에서 끝납니다. Claude Code는 이 흐름을 끊지 않고 자연스럽게 통합됩니다.

<CodeBlock
  code={`# 기본 실행
claude

# 비대화형 모드 (스크립트, 파이프라인용)
claude -p "package.json의 의존성을 분석해줘"

# 이전 세션 이어서
claude --continue

# 특정 세션 복원
claude --resume

# 모델 지정
claude --model opus

# 웹으로 세션 전송 (& prefix)
& Fix the login bug in auth.ts

# 웹 세션으로 원격 실행
claude --remote "Fix the bug in user-service.ts"`}
  language="bash"
  filename="Terminal CLI 기본 사용법"
/>

### CLI의 강점

| 강점 | 설명 |
|------|------|
| **파일 시스템 직접 접근** | 로컬 파일을 읽고, 쓰고, 실행할 수 있음 |
| **Git 네이티브 통합** | 커밋, 브랜치, PR 생성까지 자연스럽게 수행 |
| **파이프라인 호환** | stdin/stdout으로 다른 CLI 도구와 조합 가능 |
| **셸 환경 활용** | 환경 변수, 셸 스크립트, tmux 등과 통합 |
| **오프라인 파일 접근** | 네트워크 없이도 로컬 파일 탐색 가능 |

<Callout type="tip" title="터미널 CLI는 모든 플랫폼의 기반">
  VS Code 확장, JetBrains 플러그인, Desktop App 모두 내부적으로 CLI를 호출합니다.
  CLI를 먼저 숙달하면 다른 플랫폼에서도 동일한 개념을 적용할 수 있습니다.
</Callout>

## 2. Claude Code Web (claude.ai/code) — 브라우저에서 비동기 실행

### 개요

**Claude Code Web**은 브라우저에서 Claude Code의 에이전트 기능을 사용할 수 있는 서비스입니다. GitHub 저장소에 연결하여, 로컬 환경 설치 없이 코딩 태스크를 실행하고, 변경사항을 리뷰하며, PR을 생성할 수 있습니다.

<MermaidDiagram
  chart={`flowchart LR
    A[사용자] --> B[claude.ai/code]
    B --> C{GitHub 연동}
    C --> D[저장소 클론]
    D --> E[격리된 클라우드 VM]
    E --> F[코드 수정]
    E --> G[테스트 실행]
    E --> H[빌드 검증]
    F --> I[Diff 뷰로 리뷰]
    G --> I
    H --> I
    I --> J{승인?}
    J -->|Yes| K[PR 자동 생성]
    J -->|No| L[추가 수정 요청]
    L --> E
    K --> M[GitHub에 PR 등록]
  `}

### 주요 기능

| 기능 | 설명 |
|------|------|
| **병렬 태스크 실행** | 여러 작업을 동시에 실행하고 각각의 진행 상태를 모니터링 |
| **Diff 뷰** | 변경사항을 파일별로 확인하고, 라인 단위로 승인/거부 |
| **로컬 없는 리포 작업** | 로컬에 클론하지 않은 저장소에서도 작업 가능 |
| **세션 공유** | 작업 세션의 URL을 팀원에게 공유하여 협업 |
| **iOS 앱 모니터링** | 모바일에서 실행 중인 태스크 상태 확인 가능 |

### 클라우드 환경 상세

Claude Code Web은 격리된 클라우드 VM에서 실행됩니다. 각 세션마다 독립된 환경이 생성되어 보안과 격리성이 보장됩니다.

<CodeBlock
  code={`# 클라우드 환경 특성

# 1. 격리된 VM
# - 각 세션마다 독립된 가상 머신 할당
# - 세션 간 파일 시스템, 프로세스 완전 격리
# - 세션 종료 시 환경 자동 정리

# 2. 네트워크 접근 제어
# - 기본적으로 npm, pip 등 패키지 레지스트리 접근 허용
# - 외부 API 호출은 설정에 따라 제어
# - 프라이빗 네트워크 접근 불가 (보안 격리)

# 3. SessionStart Hooks로 의존성 자동 설치
# .claude/settings.json에 설정:
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "npm install"
          },
          {
            "type": "command",
            "command": "pip install -r requirements.txt"
          }
        ]
      }
    ]
  }
}`}
  language="json"
  filename="클라우드 환경 설정"
/>

<Callout type="warning" title="클라우드 환경 제약사항">
  클라우드 VM은 격리된 환경이므로 로컬 네트워크의 서비스(DB, Redis 등)에 접근할 수 없습니다.
  Docker Compose로 구성된 로컬 개발 환경이 필요한 작업은 터미널 CLI를 사용하세요.
  SessionStart Hooks를 활용하면 클라우드 환경에 필요한 의존성을 자동으로 설치할 수 있습니다.
</Callout>

### 터미널 ↔ 웹 세션 이동 (Teleport)

Claude Code의 가장 강력한 기능 중 하나는 **세션 이동(Teleport)** 입니다. 터미널에서 시작한 세션을 웹으로 전송하거나, 웹에서 실행 중인 세션을 터미널로 가져올 수 있습니다.

<MermaidDiagram
  chart={`flowchart TB
    subgraph TERMINAL["터미널 (로컬)"]
      T1["작업 시작\nclaude"]
      T2["웹으로 전송\n& prefix 또는 --remote"]
      T3["웹 세션 가져오기\n/teleport 또는 --teleport"]
    end
    subgraph WEB["Claude Code Web (클라우드)"]
      W1["태스크 실행 중"]
      W2["Diff 뷰 리뷰"]
      W3["PR 생성"]
    end
    T1 --> T2
    T2 -->|세션 전송| W1
    W1 --> W2
    W2 --> W3
    W1 -->|세션 가져오기| T3
    T3 --> T1

    style T2 fill:#e8f4e8,stroke:#4caf50
    style T3 fill:#e3f2fd,stroke:#2196f3
  `}
  title="터미널 ↔ 웹 세션 이동 (Teleport)"
  caption="세션을 터미널과 웹 사이에서 자유롭게 이동시킬 수 있습니다"
/>

<CodeBlock
  code={`# === 터미널 → 웹으로 세션 전송 ===

# 방법 1: & prefix 사용 (대화 중)
# 대화 입력 시 & 를 앞에 붙이면 해당 작업이 웹으로 전송됩니다
& Fix the authentication bug in src/auth/jwt.ts

# 방법 2: --remote 플래그 (CLI에서 직접)
claude --remote "Fix the bug in user-service.ts"
claude --remote "Add tests for the payment module"

# 여러 태스크를 병렬로 웹에 전송
claude --remote "Fix bug #42"
claude --remote "Add unit tests for UserService"
claude --remote "Update the README with API docs"


# === 웹 → 터미널로 세션 가져오기 ===

# 방법 1: /teleport 슬래시 커맨드 (대화 중)
/teleport
# 또는 줄여서
/tp

# 방법 2: --teleport 플래그 (CLI에서)
claude --teleport

# 실행 중인 웹 세션 목록에서 선택하여 로컬로 가져옵니다
# 웹에서 생성된 변경사항이 로컬에 반영됩니다`}
  language="bash"
  filename="Teleport: 터미널 ↔ 웹 세션 이동"
/>

<Callout type="tip" title="Teleport 활용 시나리오">
  **출퇴근 시나리오**: 사무실에서 터미널로 작업하다가 퇴근 전에 `& 나머지 리팩토링 완료해줘`로 웹에 전송.
  출퇴근 중 스마트폰 iOS 앱에서 진행 상태를 모니터링하고, 다음 날 `/teleport`로 결과를 터미널에 가져와서 리뷰합니다.
</Callout>

## 3. Desktop App — 독립 실행형 앱

## 4. VS Code — 네이티브 확장

### 개요

Anthropic이 직접 개발한 **공식 VS Code 확장**입니다. VS Code의 네이티브 기능과 깊이 통합되어, 인라인 diff, @멘션 파일 참조, 체크포인트 시스템 등을 제공합니다.

### 주요 기능

<CodeBlock
  code={`# VS Code 확장 설치
code --install-extension anthropic.claude-code

# 주요 기능:

# 1. 인라인 Diff
# Claude가 파일을 수정하면 VS Code의 네이티브 diff 뷰에서
# 변경사항을 확인하고 수락/거부할 수 있습니다.

# 2. @멘션 파일 참조
# 대화 중 @filename.ts로 특정 파일을 참조하면
# 해당 파일의 내용이 자동으로 컨텍스트에 포함됩니다.
# 예: "@src/services/auth.ts 이 파일의 버그를 찾아줘"

# 3. 에디터 컨텍스트 연동
# 현재 열려 있는 파일, 선택 영역, 커서 위치를
# Claude에게 자동으로 전달합니다.

# 4. 체크포인트 시스템
# Esc 두 번 또는 /rewind로 이전 상태로 복원 가능
# Claude의 변경을 실험적으로 적용하고 쉽게 되돌릴 수 있습니다.

# 5. 멀티 탭/창 지원
# 여러 Claude Code 세션을 동시에 운영 가능

# 6. Problems 패널 연동
# VS Code의 Problems 패널(린트 에러, 타입 에러)을
# Claude에게 직접 전달하여 수정 요청 가능`}
  language="bash"
  filename="VS Code Claude Code 확장"
/>

## 6. GitHub Actions — CI/CD 자동화

### 개요

Claude Code를 **GitHub Actions 워크플로우**에 통합하여, PR 리뷰, 코드 생성, 이슈 해결, 테스트 작성 등을 자동화할 수 있습니다. 이벤트 기반으로 Claude Code가 자동 실행되어, 개발자의 수동 개입 없이 반복 작업을 처리합니다.

<CodeBlock
  code={`# .github/workflows/claude-review.yml
# PR이 열리면 Claude Code가 자동으로 코드 리뷰

name: Claude Code Review
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Claude Code
        run: curl -fsSL https://claude.ai/install.sh | bash

      - name: Run Code Review
        env:
          ANTHROPIC_API_KEY: \${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          # PR의 변경사항을 Claude Code로 리뷰
          git diff origin/main...HEAD | claude -p "
            이 PR의 변경사항을 리뷰해줘.
            다음 관점으로 검토해:
            1. 보안 취약점
            2. 성능 이슈
            3. 코딩 컨벤션 위반
            4. 잠재적 버그
            결과를 마크다운으로 출력해.
          " --output-format text > review.md

      - name: Post Review Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const review = fs.readFileSync('review.md', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: review
            });`}
  language="yaml"
  filename="GitHub Actions: 자동 코드 리뷰"
/>

### 이슈 자동 해결 워크플로우

<CodeBlock
  code={`# .github/workflows/claude-issue-fix.yml
# 특정 라벨이 붙은 이슈를 Claude Code가 자동으로 해결

name: Claude Issue Fix
on:
  issues:
    types: [labeled]

jobs:
  fix:
    if: github.event.label.name == 'claude-fix'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Install Claude Code
        run: curl -fsSL https://claude.ai/install.sh | bash

      - name: Fix Issue
        env:
          ANTHROPIC_API_KEY: \${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          ISSUE_BODY="\${{ github.event.issue.body }}"
          ISSUE_TITLE="\${{ github.event.issue.title }}"

          claude -p "
            다음 이슈를 분석하고 수정해줘:
            제목: $ISSUE_TITLE
            내용: $ISSUE_BODY

            수정 후 변경된 파일 목록을 알려줘.
          " --max-turns 20

      - name: Create PR
        env:
          GH_TOKEN: \${{ secrets.GITHUB_TOKEN }}
        run: |
          ISSUE_NUM="\${{ github.event.issue.number }}"
          BRANCH="claude-fix/issue-$ISSUE_NUM"
          git checkout -b "$BRANCH"
          git add -A
          git commit -m "fix: resolve issue #$ISSUE_NUM"
          git push -u origin "$BRANCH"
          gh pr create \\
            --title "fix: resolve issue #$ISSUE_NUM" \\
            --body "Closes #$ISSUE_NUM" \\
            --base main`}
  language="yaml"
  filename="GitHub Actions: 이슈 자동 해결"
/>

<Callout type="info" title="GitHub Actions 보안 주의">
  CI/CD에서 Claude Code를 사용할 때는 반드시 `--max-turns` 플래그로 최대 실행 턴을 제한하세요.
  API 키는 GitHub Secrets에 저장하고, 절대 워크플로우 파일에 하드코딩하지 마세요.
  `--allowedTools` 플래그로 허용할 도구를 제한하면 보안을 강화할 수 있습니다.
</Callout>

## 7. GitLab CI/CD — MR/이슈 자동화

### 개요

GitLab의 Merge Request(MR)와 이슈에 대해 **이벤트 기반 자동화**를 지원합니다. GitHub Actions와 유사한 방식으로, MR이 열리면 코드 리뷰를 수행하거나, 이슈가 등록되면 자동으로 수정을 시도합니다.

<CodeBlock
  code={`# .gitlab-ci.yml
# MR이 열리면 Claude Code가 자동으로 코드 리뷰

stages:
  - review

claude-review:
  stage: review
  image: ubuntu:latest
  rules:
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
  before_script:
    - curl -fsSL https://claude.ai/install.sh | bash
  script:
    - |
      # MR의 변경사항을 Claude Code로 리뷰
      git diff origin/$CI_MERGE_REQUEST_TARGET_BRANCH_NAME...HEAD | claude -p "
        이 MR의 변경사항을 리뷰해줘.
        보안, 성능, 버그 관점에서 검토하고 마크다운으로 결과를 출력해.
      " --output-format text > review.md

      # GitLab MR에 코멘트 작성
      curl --request POST \\
        --header "PRIVATE-TOKEN: $GITLAB_TOKEN" \\
        --header "Content-Type: application/json" \\
        --data "{\"body\": \"$(cat review.md | jq -Rs .)\"}" \\
        "$CI_API_V4_URL/projects/$CI_PROJECT_ID/merge_requests/$CI_MERGE_REQUEST_IID/notes"
  variables:
    ANTHROPIC_API_KEY: $ANTHROPIC_API_KEY
    GITLAB_TOKEN: $GITLAB_TOKEN`}
  language="yaml"
  filename="GitLab CI/CD: MR 자동 리뷰"
/>

### GitLab 이슈 자동 처리

<CodeBlock
  code={`# .gitlab-ci.yml (이슈 자동 처리 추가)

claude-issue-fix:
  stage: review
  image: ubuntu:latest
  rules:
    - if: '$CI_PIPELINE_SOURCE == "trigger"'
      when: always
  before_script:
    - curl -fsSL https://claude.ai/install.sh | bash
    - apt-get update && apt-get install -y git jq
  script:
    - |
      # 이슈 정보 조회
      ISSUE_DATA=$(curl -s --header "PRIVATE-TOKEN: $GITLAB_TOKEN" \\
        "$CI_API_V4_URL/projects/$CI_PROJECT_ID/issues/$ISSUE_IID")

      ISSUE_TITLE=$(echo "$ISSUE_DATA" | jq -r '.title')
      ISSUE_DESC=$(echo "$ISSUE_DATA" | jq -r '.description')

      # Claude Code로 수정
      claude -p "
        GitLab 이슈를 분석하고 수정해줘:
        제목: $ISSUE_TITLE
        내용: $ISSUE_DESC
      " --max-turns 15

      # MR 생성
      BRANCH="claude-fix/issue-$ISSUE_IID"
      git checkout -b "$BRANCH"
      git add -A
      git commit -m "fix: resolve issue #$ISSUE_IID"
      git push -u origin "$BRANCH"

      # GitLab MR 생성
      curl --request POST \\
        --header "PRIVATE-TOKEN: $GITLAB_TOKEN" \\
        --header "Content-Type: application/json" \\
        --data "{
          \"source_branch\": \"$BRANCH\",
          \"target_branch\": \"main\",
          \"title\": \"fix: resolve issue #$ISSUE_IID\",
          \"description\": \"Closes #$ISSUE_IID\"
        }" \\
        "$CI_API_V4_URL/projects/$CI_PROJECT_ID/merge_requests"
  variables:
    ANTHROPIC_API_KEY: $ANTHROPIC_API_KEY
    GITLAB_TOKEN: $GITLAB_TOKEN`}
  language="yaml"
  filename="GitLab CI/CD: 이슈 자동 해결"
/>

## 자주 묻는 질문

### 플랫폼 간 세션 호환성

**Q: 터미널에서 시작한 세션을 VS Code 확장에서 이어서 할 수 있나요?**

A: 터미널 CLI와 VS Code 확장은 별도의 세션 공간을 사용합니다. 다만 `claude --resume`으로 이전 CLI 세션을 복원하거나, `--continue`로 가장 최근 세션을 이어갈 수 있습니다. 터미널 ↔ 웹 간 세션 이동은 Teleport로 가능합니다.

**Q: Claude Code Web에서 프라이빗 npm 레지스트리를 사용할 수 있나요?**

A: 클라우드 환경은 격리되어 있어 기본적으로 프라이빗 레지스트리에 접근할 수 없습니다. SessionStart Hooks에서 `.npmrc` 설정을 추가하거나, 필요한 패키지를 public으로 전환하는 방법을 고려하세요.

**Q: GitHub Actions에서 Claude Code의 비용은 어떻게 관리하나요?**

A: API 키 사용 시 종량제로 과금됩니다. `--max-turns`로 실행 턴을 제한하고, Anthropic Console에서 일일 예산을 설정하세요. 또한 CI/CD에서는 Sonnet 모델을 기본으로 사용하여 비용을 최적화하는 것을 권장합니다.

**Q: Slack에서 Claude가 실행하는 작업의 결과를 어디서 확인하나요?**

A: Slack 스레드에 진행 상황과 결과 PR 링크가 자동으로 게시됩니다. 상세한 Diff는 GitHub/GitLab의 PR 페이지에서 확인할 수 있습니다.

## 요약

Claude Code는 **9개 플랫폼**에서 실행되며, 각 플랫폼은 동일한 에이전트 아키텍처를 공유합니다.

| 플랫폼 | 핵심 역할 | 최적 사용 시점 |
|--------|----------|---------------|
| **Terminal CLI** | 핵심 인터페이스, 모든 플랫폼의 기반 | 일상적 코딩, 깊은 디버깅 |
| **Claude Code Web** | 브라우저에서 비동기 실행 | 모바일 작업, 설치 없는 환경, 병렬 태스크 |
| **Desktop App** | 독립 앱, Diff 리뷰, worktree | 병렬 브랜치 작업, 시각적 리뷰 |
| **VS Code** | 네이티브 IDE 통합 | VS Code 사용자의 일상 개발 |
| **JetBrains** | IDE 플러그인 | JetBrains 사용자의 일상 개발 |
| **GitHub Actions** | CI/CD 자동화 | PR 리뷰, 이슈 해결 자동화 |
| **GitLab CI/CD** | MR/이슈 자동화 | GitLab 기반 프로젝트 |
| **Slack** | 대화형 태스크 전달 | 팀 협업, 빠른 태스크 할당 |
| **Chrome** | 브라우저 제어 (Beta) | UI 디버깅, E2E 테스트 |

핵심은 **세션 이동(Teleport)** 입니다. 터미널에서 시작한 작업을 웹으로 전송하고, 모바일에서 모니터링하고, 다시 터미널로 가져올 수 있습니다. 개발자가 어디에 있든, 어떤 디바이스를 사용하든, Claude Code와의 작업은 끊김 없이 이어집니다.

<Callout type="tip" title="플랫폼은 수단, 목표는 생산성">
  모든 플랫폼을 사용할 필요는 없습니다.
  자신의 워크플로우에 맞는 1-2개 플랫폼으로 시작하고, 필요에 따라 확장하세요.
  대부분의 개발자에게는 **Terminal CLI + IDE 확장 + Claude Code Web** 조합이 가장 효과적입니다.
</Callout>
