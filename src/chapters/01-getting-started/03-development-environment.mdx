## IDE 통합

Claude Code는 터미널 CLI가 기본이지만, 주요 IDE와의 공식 통합을 제공합니다.

### VS Code 확장

Anthropic이 직접 개발하고 배포하는 공식 확장입니다.

<CodeBlock
  code={`# 설치 방법
# 1. VS Code에서 Cmd+Shift+X (Mac) / Ctrl+Shift+X (Windows/Linux)
# 2. "Claude Code" 검색 → Anthropic 퍼블리셔 확인 → 설치
# 3. Claude Code CLI가 사전 설치되어 있어야 합니다

# 또는 커맨드라인에서
code --install-extension anthropic.claude-code`}
  language="bash"
  filename="VS Code 확장 설치"

**주요 기능:**
- 편집 전 변경 계획 검토
- 자동 편집 수락 (수동 확인 불필요)
- `@` 멘션으로 파일 참조
- 대화 히스토리 및 여러 탭/창 지원
- 실시간 파일 변경사항 확인
- 체크포인트 시스템 (Esc 두 번 또는 `/rewind`로 이전 상태 복원)

#### Neovim + Claude Code 터미널 연동

avante.nvim 외에도 Neovim의 내장 터미널에서 Claude Code CLI를 직접 사용할 수 있습니다.

<CodeBlock
  code={`-- ~/.config/nvim/lua/config/claude-terminal.lua

-- Claude Code 전용 터미널 토글
vim.keymap.set('n', '<leader>cc', function()
  vim.cmd('botright split | terminal claude')
end, { desc = 'Claude Code 터미널 열기' })

-- 현재 파일을 Claude Code에 전달
vim.keymap.set('n', '<leader>cf', function()
  local file = vim.fn.expand('%:p')
  vim.cmd('botright split | terminal claude "이 파일을 검토해줘: ' .. file .. '"')
end, { desc = '현재 파일 Claude Code로 전달' })

-- 선택 영역을 Claude Code에 전달
vim.keymap.set('v', '<leader>cs', function()
  -- 선택된 텍스트 복사 후 Claude Code에 전달
  vim.cmd('normal! y')
  local text = vim.fn.getreg('"')
  vim.cmd('botright split | terminal claude')
end, { desc = '선택 영역 Claude Code로 전달' })`}
  language="lua"
  filename="Neovim Claude Code 키맵 설정"
/>

<Callout type="tip" title="Vim 모드 내장 지원">
  Claude Code CLI 자체에 `/vim` 명령으로 Vim 모드가 내장되어 있습니다.
  별도 플러그인 없이도 Vim 스타일의 입력 편집이 가능합니다.
  Insert 모드와 Normal 모드를 지원하며, `Esc`로 모드를 전환합니다.
</Callout>

### tmux 연동

tmux를 사용하면 세션을 유지하면서 여러 Claude Code 인스턴스를 관리할 수 있습니다.

<CodeBlock
  code={`# ~/.tmux.conf - Claude Code 최적화 설정

# 24비트 색상 지원
set -g default-terminal "tmux-256color"
set -ag terminal-overrides ",xterm-256color:RGB"

# 마우스 지원
set -g mouse on

# 스크롤백 버퍼 증가 (Claude Code 출력이 길 수 있음)

# 상태 표시줄 커스터마이징
set -g status-right '#[fg=green]#{pane_current_path} #[fg=yellow]%H:%M'

# 창 분할 단축키 (직관적)
bind | split-window -h -c "#{pane_current_path}"
bind - split-window -v -c "#{pane_current_path}"

# 새 창을 현재 디렉토리에서 열기
bind c new-window -c "#{pane_current_path}"

# Vim 스타일 패인 이동
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 패인 크기 조절

bind -r J resize-pane -D 5
bind -r K resize-pane -U 5

#### tmux + Claude Code 워크플로우

<CodeBlock
  code={`# Claude Code 개발 세션 스크립트
# ~/scripts/claude-dev-session.sh

#!/bin/bash
SESSION="claude-dev"

# 세션 존재 여부 확인
tmux has-session -t $SESSION 2>/dev/null

if [ $? != 0 ]; then
  # 새 세션 생성: 왼쪽에 에디터, 오른쪽에 Claude Code
  tmux new-session -d -s $SESSION -n 'editor'

  # 왼쪽 패인: 코드 편집 (vim 또는 에디터)
  tmux send-keys -t $SESSION 'vim .' C-m

  # 오른쪽 패인: Claude Code
  tmux split-window -h -t $SESSION
  tmux send-keys -t $SESSION 'claude' C-m

  # 하단 패인: 일반 터미널 (테스트, git 등)
  tmux split-window -v -t $SESSION
  tmux send-keys -t $SESSION 'echo "Ready for testing"' C-m

  # 레이아웃 조정 (에디터 60%, Claude 40%)
  tmux select-layout -t $SESSION main-vertical
fi

# 세션 연결
tmux attach-session -t $SESSION`}
  language="bash"
  filename="Claude Code tmux 개발 세션 스크립트"
/>

### 키바인딩

Claude Code의 주요 키바인딩입니다.

| 키 | 기능 | 비고 |
|-----|------|------|
| **Enter** | 메시지 전송 | |
| **Shift+Enter** | 줄바꿈 (멀티라인) | iTerm2, WezTerm, Ghostty, Kitty는 기본 지원. 그 외 `/terminal-setup` 실행 필요 |
| **Escape (2회)** | 이전 메시지로 되돌리기 (rewind) | |
| **Shift+Tab (2회)** | Plan Mode 전환 | 파일 수정 없이 탐색/계획만 |
| **Tab** | 파일/경로 자동완성 | |

<Callout type="warning" title="Shift+Enter 설정 필요">
  Shift+Enter 줄바꿈은 iTerm2, WezTerm, Ghostty, Kitty에서는 기본 지원됩니다.

## 환경 설정 체크리스트

설정을 마친 후 다음 항목을 확인하세요.

| 항목 | 확인 방법 | 기대 결과 |
|------|-----------|-----------|
| CLI 설치 | `claude --version` | 버전 번호 출력 |
| API 키 | `claude "안녕"` | 정상 응답 |
| Shift+Enter | 터미널에서 Shift+Enter | 줄바꿈 |
| IDE 확장 | 확장 패널 확인 | Claude Code 활성화 |
| 설정 파일 | `/config` 실행 | 설정 목록 표시 |
| .claudeignore | `claude` 실행 시 속도 | node_modules 등 제외 |
| CLAUDE.md | `/init` 또는 직접 작성 | 프로젝트 규칙 로드 |

<Callout type="info" title="문제 해결">
  환경 설정 중 문제가 발생하면 [Claude Code 트러블슈팅 문서](https://code.claude.com/docs/en/troubleshooting)를 참조하세요.
  `claude --verbose` 플래그를 사용하면 상세한 디버그 로그를 확인할 수 있습니다.
</Callout>
