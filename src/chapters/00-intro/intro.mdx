# Vibe Coding & Claude Code 교육

> AI Agent와 함께하는 현대적 소프트웨어 개발의 새로운 패러다임

## 이 교육에서 배우는 것

이 교육 자료는 **Vibe Coding**의 개념부터 **Claude Code**를 활용한 실전 개발 워크플로우까지 체계적으로 다룹니다.

<Callout type="info" title="교육 대상">
  AI 도구를 활용한 개발에 관심있는 모든 개발자를 대상으로 합니다.
  기본적인 프로그래밍 경험이 있으면 충분합니다.
</Callout>

## 교육 구성

| Part | 주제 | 내용 |
|------|------|------|
| **Part 1** | 기초 | 바이브 코딩, Claude Code, CLAUDE.md, 엔터프라이즈 보안, 프롬프트 엔지니어링 |
| **Part 2** | 프로젝트 템플릿 구조 | 프로젝트 지식 구조, 컨텍스트, 아키텍처, 가이드라인 |
| **Part 3** | AI Agent 워크플로우 | 슬래시 커맨드, 프롬프트, 스킬, 서브에이전트, Hooks |
| **Part 4** | MCP와 통합 | Model Context Protocol, GitHub Actions, MCP 서버 활용 |
| **Part 5** | 실전 예제 | TypeScript API, Python API, 프론트엔드, E2E 워크플로우 |
| **Part 6** | 모범 사례 | 보안, 팀 협업, 효과 측정 |

<Callout type="info" title="2026년 2월 업데이트">
  이 교육 자료는 **Claude Opus 4.6**, **Agent Teams**, **Cowork**, **Claude Agent SDK** 등 2026년 2월 기준 최신 기능을 포함합니다.
</Callout>

## 이 교육의 특별한 점

<Callout type="info" title="왜 이 교육인가?">
  시중에는 다양한 AI 코딩 교육이 있지만, 이 교육은 다음과 같은 차별점을 가집니다:

  **1. 엔터프라이즈 관점의 실전 가이드**
  단순히 "AI에게 코드 생성 시키기"가 아닌, 팀 환경에서의 보안, 거버넌스, 코드 품질 관리까지 포괄합니다.
  실제 기업 환경에서 AI 코딩 도구를 도입할 때 마주치는 현실적인 문제들을 다룹니다.

  **2. CLAUDE.md 중심의 프로젝트 지식 관리**
  프로젝트의 맥락, 아키텍처, 컨벤션을 체계적으로 문서화하여 AI Agent가 일관된 코드를 생성하도록 하는 방법론을 제시합니다.
  이는 AI 도구의 "일회성 코드 생성"을 넘어 "지속 가능한 AI 협업"으로 이끕니다.

  **3. 2026년 2월 최신 기능 반영**
  Opus 4.6, Agent Teams, Cowork, Claude Agent SDK, MCP의 Linux Foundation 이관 등
  가장 최신의 기능과 생태계 변화를 반영합니다.

  **4. 실습 중심의 학습 구조**
  모든 챕터에 실행 가능한 코드 예제, 프롬프트 템플릿, 설정 파일 예시가 포함되어 있어
  읽는 것만으로 끝나지 않고 직접 따라하며 체득할 수 있습니다.

  **5. 역할별 맞춤 학습 경로**
  주니어 개발자, 시니어 개발자, 테크 리드, 비개발자 등 각 역할에 맞는
  추천 학습 경로를 제공하여 효율적으로 필요한 내용을 학습할 수 있습니다.
</Callout>

## Part별 상세 학습 목표

### Part 1: 기초 (8개 챕터)

이 Part를 마치면 AI 코딩의 기본 개념을 이해하고 Claude Code를 설치하여 첫 번째 대화를 나눌 수 있습니다.

| 챕터 | 학습 목표 |
|------|----------|
| 바이브 코딩이란? | Vibe Coding의 정의, 철학, 장점과 한계를 이해한다 |
| Claude Code란? | Claude Code의 아키텍처, 핵심 기능, 다른 도구와의 차이를 파악한다 |
| CLAUDE.md 딥다이브 | 프로젝트 지식을 CLAUDE.md에 체계적으로 구조화하는 방법을 익힌다 |
| 엔터프라이즈 보안 | 기업 환경에서 AI 코딩 도구의 보안 위험과 대응 전략을 학습한다 |
| 프롬프트 엔지니어링 | Claude Code에 최적화된 프롬프트 작성 기법을 연습한다 |
| 모델 선택 가이드 | 작업 유형별 최적의 Claude 모델을 선택하는 기준을 세운다 |
| 개발 환경 설정 | Claude Code의 설치, 인증, 초기 설정을 완료한다 |
| 법적/윤리적 고려사항 | AI 생성 코드의 라이선스, 저작권, 윤리적 이슈를 인식한다 |

<Callout type="tip" title="Part 1 학습 팁">
  Part 1은 모든 학습자에게 공통으로 권장됩니다. 특히 "CLAUDE.md 딥다이브"와 "프롬프트 엔지니어링" 챕터는
  이후 모든 Part의 기반이 되므로 반드시 숙지하세요.
</Callout>

### Part 2: 프로젝트 템플릿 구조 (6개 챕터)

이 Part를 마치면 AI Agent가 프로젝트 맥락을 이해할 수 있도록 지식 구조를 설계하고 문서화할 수 있습니다.

| 챕터 | 학습 목표 |
|------|----------|
| 에이전트 디렉토리 개요 | `.claude/` 디렉토리의 구조와 각 파일의 역할을 이해한다 |
| 컨텍스트와 컨벤션 | 프로젝트 컨텍스트 문서와 코딩 컨벤션 파일을 작성한다 |
| 아키텍처 | 시스템 아키텍처를 AI가 이해할 수 있는 형태로 문서화한다 |
| 가이드라인 | AI Agent 사용 가이드라인과 코드 리뷰 규칙을 수립한다 |
| 문서화 패턴 | 효과적인 프로젝트 문서화의 패턴과 안티패턴을 구분한다 |
| 고급 패턴 | 멀티 저장소 설정, 조건부 규칙 등 고급 구성을 활용한다 |

### Part 3: AI Agent 워크플로우 (5개 챕터)

이 Part를 마치면 Claude Code의 고급 기능을 활용하여 개발 워크플로우를 자동화할 수 있습니다.

| 챕터 | 학습 목표 |
|------|----------|
| 슬래시 커맨드 | `/init`, `/review`, `/commit` 등 내장 커맨드를 활용한다 |
| 프롬프트 라이브러리 | 재사용 가능한 프롬프트를 `.claude/prompts/`에 구성한다 |
| 스킬 | 커스텀 스킬을 정의하고 `context:fork`, Lifecycle Hooks를 활용한다 |
| 서브에이전트 | Agent Teams를 활용한 멀티 에이전트 협업 패턴을 구현한다 |
| Hooks | Pre/Post Hook으로 CI/CD 파이프라인과 연동한다 |

### Part 4: MCP와 통합 (3개 챕터)

이 Part를 마치면 Model Context Protocol을 이해하고 외부 도구 및 서비스와 통합할 수 있습니다.

| 챕터 | 학습 목표 |
|------|----------|
| MCP 개요 | MCP의 아키텍처, 프로토콜, 생태계를 이해한다 |
| GitHub Actions | Claude Code를 CI/CD 파이프라인에 통합한다 |
| MCP 서버 활용 | 데이터베이스, API, 파일 시스템 MCP 서버를 설정하고 사용한다 |

### Part 5: 실전 예제 (4개 챕터)

이 Part를 마치면 실제 프로젝트를 Claude Code와 함께 처음부터 끝까지 구축할 수 있습니다.

| 챕터 | 학습 목표 |
|------|----------|
| TypeScript API | Express.js 기반 REST API를 Claude Code로 구축한다 |
| Python API | FastAPI 기반 백엔드를 Claude Code로 구축한다 |
| 프론트엔드 프로젝트 | React 프론트엔드를 Claude Code로 구축한다 |
| E2E 워크플로우 | 기획부터 배포까지 전체 개발 사이클을 AI와 함께 수행한다 |

### Part 6: 모범 사례 (3개 챕터)

이 Part를 마치면 팀과 조직 차원에서 AI 코딩 도구를 안전하고 효과적으로 도입/운영할 수 있습니다.

| 챕터 | 학습 목표 |

## 자주 묻는 질문 (FAQ)

### 일반 질문

**Q: Vibe Coding이 기존 코딩을 완전히 대체하나요?**

아닙니다. Vibe Coding은 기존 프로그래밍 능력을 **증폭(amplify)**시키는 도구입니다. 알고리즘적 사고, 시스템 설계, 디버깅 능력 등 핵심 개발 역량은 여전히 중요합니다. AI는 반복적인 작업을 줄여주고, 새로운 기술 학습 곡선을 낮춰주며, 코드 리뷰를 보조하는 역할을 합니다. 최종 코드의 품질과 정확성에 대한 판단은 개발자의 몫입니다.

**Q: Claude Code를 사용하려면 유료 구독이 필요한가요?**

Claude Code는 **Claude Pro($20/월)**, **Claude Max($100~200/월)**, 또는 **Anthropic API 키**로 사용할 수 있습니다. Pro 플랜은 일일 사용량 제한이 있고, Max 플랜은 더 넉넉한 사용량을 제공합니다. API 키 방식은 사용한 만큼만 과금되므로 팀이나 CI/CD 환경에 적합합니다.

**Q: AI가 생성한 코드의 보안은 어떻게 보장하나요?**

이 교육의 Part 1(엔터프라이즈 보안)과 Part 6(보안 모범 사례)에서 이 주제를 깊이 다룹니다. 핵심은 **AI 생성 코드를 무조건 신뢰하지 않는 것**입니다. 기존의 코드 리뷰 프로세스, SAST/DAST 도구, 의존성 스캔을 AI 생성 코드에도 동일하게 적용해야 합니다.

### 기술적 질문

**Q: Claude Code와 GitHub Copilot의 차이점은 무엇인가요?**

GitHub Copilot은 주로 에디터 내에서 **코드 자동완성**에 초점을 맞추는 반면, Claude Code는 **터미널 기반의 에이전트**로서 파일 생성/수정, Git 작업, 테스트 실행, 리팩토링 등 더 넓은 범위의 작업을 자율적으로 수행합니다. 두 도구는 경쟁 관계보다 **보완 관계**에 가깝습니다.

**Q: 어떤 프로그래밍 언어를 지원하나요?**

Claude Code는 특정 언어에 제한되지 않습니다. TypeScript, Python, Go, Rust, Java, C/C++, Ruby, PHP 등 주요 언어는 물론이고, Terraform, SQL, YAML 등 설정 파일도 잘 다룹니다. 다만 이 교육에서는 주로 **TypeScript**와 **Python** 예제를 사용합니다.

**Q: 오프라인에서도 사용할 수 있나요?**

아닙니다. Claude Code는 Anthropic의 API 서버와 통신해야 하므로 **인터넷 연결이 필수**입니다. 다만, CLAUDE.md와 같은 프로젝트 지식 파일은 로컬에 저장되므로 오프라인에서 편집할 수 있고, 다시 온라인이 되면 Claude Code가 이를 읽어 활용합니다.

**Q: 회사 코드가 Anthropic 서버로 전송되나요?**

Claude Code는 대화 중 필요한 코드 스니펫을 API로 전송합니다. 엔터프라이즈 환경에서는 Anthropic의 데이터 보존 정책을 확인하고, 필요시 API 키 방식에서의 **제로 데이터 보존(Zero Data Retention)** 옵션을 활용하세요. 자세한 보안 고려사항은 Part 1의 엔터프라이즈 보안 챕터에서 다룹니다.

### 학습 관련 질문

**Q: 이 교육을 완료하는 데 얼마나 걸리나요?**

모든 챕터를 실습과 함께 학습하면 약 **25-35시간** 정도 소요됩니다. 역할별 추천 경로를 따르면 핵심 내용을 **5-20시간** 내에 학습할 수 있습니다. 한번에 다 끝내려 하기보다 매일 1-2챕터씩 꾸준히 진행하는 것을 권장합니다.

**Q: 실습 환경은 어떻게 준비하나요?**

아래 "실습 준비" 섹션의 설치 가이드를 따르세요. macOS, Linux, Windows 모두 지원됩니다. 특별한 하드웨어 요구사항은 없으며, 일반적인 개발 환경이면 충분합니다.

**Q: 교육 내용이 업데이트되면 어떻게 알 수 있나요?**

이 교육 자료는 Git 저장소에서 관리되며, Claude Code와 AI 코딩 생태계의 변화에 맞춰 지속적으로 업데이트됩니다. 저장소를 watch하거나 주기적으로 pull하여 최신 내용을 확인하세요.

## 학습 방법

### 기본 학습 원칙

1. **순서대로 학습**: Part 1부터 차례로 진행하세요. 각 Part는 이전 내용을 기반으로 합니다.
2. **실습 병행**: 코드 블록의 예제를 직접 따라해보세요. 읽기만 해서는 체득되지 않습니다.
3. **복습하기**: 각 챕터의 핵심 개념을 직접 실습해보세요. 특히 CLAUDE.md 작성과 프롬프트 작성은 반복 연습이 중요합니다.
4. **파일 뷰어 활용**: 실제 템플릿 파일 내용을 직접 확인하세요. 교육 자료 자체가 프로젝트 구조의 좋은 예시입니다.

### 효과적인 학습 전략

**능동적으로 실험하세요**

교육 예제를 그대로 따라하는 것에서 멈추지 말고, 변형해보세요. 프롬프트를 바꿔보고, 다른 언어로 시도해보고, 의도적으로 "잘못된" 프롬프트를 줘서 AI의 한계를 직접 경험해보세요.

**자신만의 CLAUDE.md를 만드세요**

교육 중 배운 내용을 바탕으로 자신의 프로젝트에 적합한 CLAUDE.md를 작성해보세요. 이 과정에서 프로젝트 구조와 컨벤션을 더 깊이 이해하게 됩니다.

**실패를 기록하세요**

AI가 잘못된 코드를 생성하거나 의도를 잘못 파악한 경우, 그 원인을 분석하고 기록해두세요. "어떤 프롬프트가 왜 실패했는지"를 이해하면 프롬프트 작성 실력이 빠르게 향상됩니다.

**팀과 함께 학습하세요**

가능하다면 팀원들과 함께 이 교육을 진행하세요. 같은 문제를 다른 프롬프트로 해결하는 과정을 공유하면 훨씬 다양한 시각을 얻을 수 있습니다.
