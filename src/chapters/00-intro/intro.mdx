# Vibe Coding & Claude Code 교육

> AI Agent와 함께하는 현대적 소프트웨어 개발의 새로운 패러다임

## 이 교육에서 배우는 것

이 교육 자료는 **Vibe Coding**의 개념부터 **Claude Code**를 활용한 실전 개발 워크플로우까지 체계적으로 다룹니다.

<Callout type="info" title="교육 대상">
  AI 도구를 활용한 개발에 관심있는 모든 개발자를 대상으로 합니다.
  기본적인 프로그래밍 경험이 있으면 충분합니다.
</Callout>

## 교육 구성

| Part | 주제 | 내용 |
|------|------|------|
| **Part 1** | 시작하기 | 바이브 코딩, Claude Code, 개발 환경 설정, 모델 선택 가이드 |
| **Part 2** | 프로젝트 설정 | CLAUDE.md 딥다이브, 프로젝트 지식 구조, 아키텍처, 가이드라인, 문서 템플릿 |

| **Part 4** | 확장과 자동화 | MCP 개요, MCP 서버 활용, CI/CD 통합, Agent SDK |
| **Part 5** | 실전 예제 | TypeScript API, Python API, 프론트엔드, E2E 워크플로우 |
| **Part 6** | 거버넌스 | 보안, 법적 윤리적 고려사항, 팀 협업, 효과 측정 |

<Callout type="info" title="2026년 2월 업데이트">

## 2026년 AI 코딩 시장 현황과 전망

AI 코딩 도구 시장은 2025년을 기점으로 폭발적 성장기에 진입했습니다. 단순한 코드 자동완성을 넘어, 자율적으로 소프트웨어를 설계하고 구축하는 **에이전트 기반 개발**이 주류가 되고 있습니다.

### 시장 규모와 성장

- **2025년 글로벌 AI 코딩 도구 시장**: 리서치 기관에 따라 $74억~$151억 규모로 추정 ([Mordor Intelligence](https://www.mordorintelligence.com/industry-reports/artificial-intelligence-code-tools-market), [Market Research Future](https://www.marketresearchfuture.com/reports/ai-code-tool-market-28659))
- **개발자 채택률**: [Stack Overflow 2025 Developer Survey](https://survey.stackoverflow.co/2025/ai)에 따르면 전 세계 개발자의 약 78%가 AI 코딩 도구를 사용 중 (사용 또는 계획 중 84%)
- **엔터프라이즈 도입률**: Fortune 100 기업의 90%가 GitHub Copilot을 도입 ([TechCrunch, 2025.07](https://techcrunch.com/2025/07/30/github-copilot-crosses-20-million-all-time-users/))

### 핵심 트렌드

| 트렌드 | 설명 | 영향 |
|--------|------|------|
| **에이전트형 AI** | 단순 자동완성에서 자율적 작업 수행 에이전트로 진화 | 개발 워크플로우 전체를 AI가 오케스트레이션 |
| **멀티 에이전트 협업** | 여러 AI 에이전트가 역할을 분담하여 협력 | 대규모 프로젝트의 병렬 개발 가능 |
| **프로토콜 표준화** | MCP의 Linux Foundation 이관으로 업계 표준화 가속 | 도구 간 호환성 및 생태계 확대 |
| **코드 안전성 강화** | AI가 보안 취약점을 사전에 탐지하고 수정 | 제로데이 취약점 발견 능력이 인간 전문가 수준 도달 |
| **비개발자의 참여** | Vibe Coding으로 비전공자도 소프트웨어 제작 가능 | 소프트웨어 개발 인구의 급격한 확대 |

### 산업별 특징

**금융/핀테크**

규제 준수(Compliance)가 최우선 과제입니다. AI 생성 코드에 대한 감사 추적(Audit Trail)이 필수이며, 코드의 결정론적 동작을 검증해야 합니다. 그러나 트레이딩 알고리즘 프로토타이핑, 리스크 모델링 보조, 규정 준수 체크 자동화 등에서 높은 효용을 보이고 있습니다.

**의료/헬스케어**

HIPAA, 의료기기 소프트웨어 규정 등으로 인해 보수적인 도입이 이루어지고 있습니다. 하지만 임상 데이터 분석 파이프라인, 의료 영상 처리 보조 코드, 전자건강기록(EHR) 시스템 연동 등에서 활용이 확대되고 있습니다.

**공공/정부**

데이터 주권과 보안 인증 요구사항으로 클라우드 기반 AI 도구의 도입이 가장 느립니다. 그러나 내부 행정 시스템 현대화, 시민 서비스 디지털화 프로젝트에서 AI 코딩 도구의 잠재력이 인식되기 시작했습니다.

**스타트업**

가장 적극적인 도입층입니다. 소규모 팀이 AI를 활용하여 대기업 수준의 개발 속도를 달성하는 사례가 증가하고 있습니다. "AI-첫 번째(AI-first)" 개발 문화를 처음부터 구축하는 스타트업도 등장하고 있습니다.

**대기업 IT**

체계적인 파일럿 프로그램을 통해 단계적으로 도입하고 있습니다. 사내 AI 코딩 가이드라인 수립, 보안 검토 프로세스 정비, 개발자 교육 프로그램 운영 등 조직적 차원의 준비를 병행합니다.

## 이 교육의 특별한 점

<Callout type="info" title="왜 이 교육인가?">
  시중에는 다양한 AI 코딩 교육이 있지만, 이 교육은 다음과 같은 차별점을 가집니다:

  **1. 엔터프라이즈 관점의 실전 가이드**
  단순히 "AI에게 코드 생성 시키기"가 아닌, 팀 환경에서의 보안, 거버넌스, 코드 품질 관리까지 포괄합니다.
  실제 기업 환경에서 AI 코딩 도구를 도입할 때 마주치는 현실적인 문제들을 다룹니다.

  **2. CLAUDE.md 중심의 프로젝트 지식 관리**
  프로젝트의 맥락, 아키텍처, 컨벤션을 체계적으로 문서화하여 AI Agent가 일관된 코드를 생성하도록 하는 방법론을 제시합니다.
  이는 AI 도구의 "일회성 코드 생성"을 넘어 "지속 가능한 AI 협업"으로 이끕니다.

  **3. 2026년 2월 최신 기능 반영**
  Opus 4.6, Agent Teams, Cowork, Claude Agent SDK, MCP의 Linux Foundation 이관 등
  가장 최신의 기능과 생태계 변화를 반영합니다.

  **4. 실습 중심의 학습 구조**
  모든 챕터에 실행 가능한 코드 예제, 프롬프트 템플릿, 설정 파일 예시가 포함되어 있어
  읽는 것만으로 끝나지 않고 직접 따라하며 체득할 수 있습니다.

  **5. 역할별 맞춤 학습 경로**
  주니어 개발자, 시니어 개발자, 테크 리드, 비개발자 등 각 역할에 맞는
  추천 학습 경로를 제공하여 효율적으로 필요한 내용을 학습할 수 있습니다.
</Callout>

## 자주 묻는 질문 (FAQ)

### 일반 질문

**Q: Vibe Coding이 기존 코딩을 완전히 대체하나요?**

아닙니다. Vibe Coding은 기존 프로그래밍 능력을 **증폭(amplify)**시키는 도구입니다. 알고리즘적 사고, 시스템 설계, 디버깅 능력 등 핵심 개발 역량은 여전히 중요합니다. AI는 반복적인 작업을 줄여주고, 새로운 기술 학습 곡선을 낮춰주며, 코드 리뷰를 보조하는 역할을 합니다. 최종 코드의 품질과 정확성에 대한 판단은 개발자의 몫입니다.

**Q: Claude Code를 사용하려면 유료 구독이 필요한가요?**

Claude Code는 **Claude Pro($20/월)**, **Claude Max($100~200/월)**, 또는 **Anthropic API 키**로 사용할 수 있습니다. Pro 플랜은 일일 사용량 제한이 있고, Max 플랜은 더 넉넉한 사용량을 제공합니다. API 키 방식은 사용한 만큼만 과금되므로 팀이나 CI/CD 환경에 적합합니다.

**Q: AI가 생성한 코드의 보안은 어떻게 보장하나요?**

이 교육의 Part 6(거버넌스)에서 보안과 법적/윤리적 고려사항을 깊이 다룹니다. 핵심은 **AI 생성 코드를 무조건 신뢰하지 않는 것**입니다. 기존의 코드 리뷰 프로세스, SAST/DAST 도구, 의존성 스캔을 AI 생성 코드에도 동일하게 적용해야 합니다.

### 기술적 질문

**Q: Claude Code와 GitHub Copilot의 차이점은 무엇인가요?**

GitHub Copilot은 주로 에디터 내에서 **코드 자동완성**에 초점을 맞추는 반면, Claude Code는 **터미널 기반의 에이전트**로서 파일 생성/수정, Git 작업, 테스트 실행, 리팩토링 등 더 넓은 범위의 작업을 자율적으로 수행합니다. 두 도구는 경쟁 관계보다 **보완 관계**에 가깝습니다.

**Q: 어떤 프로그래밍 언어를 지원하나요?**

Claude Code는 특정 언어에 제한되지 않습니다. TypeScript, Python, Go, Rust, Java, C/C++, Ruby, PHP 등 주요 언어는 물론이고, Terraform, SQL, YAML 등 설정 파일도 잘 다룹니다. 다만 이 교육에서는 주로 **TypeScript**와 **Python** 예제를 사용합니다.

**Q: 오프라인에서도 사용할 수 있나요?**

아닙니다. Claude Code는 Anthropic의 API 서버와 통신해야 하므로 **인터넷 연결이 필수**입니다. 다만, CLAUDE.md와 같은 프로젝트 지식 파일은 로컬에 저장되므로 오프라인에서 편집할 수 있고, 다시 온라인이 되면 Claude Code가 이를 읽어 활용합니다.

**Q: 회사 코드가 Anthropic 서버로 전송되나요?**

Claude Code는 대화 중 필요한 코드 스니펫을 API로 전송합니다. 엔터프라이즈 환경에서는 Anthropic의 데이터 보존 정책을 확인하고, 필요시 API 키 방식에서의 **제로 데이터 보존(Zero Data Retention)** 옵션을 활용하세요. 자세한 보안 고려사항은 Part 6의 보안 챕터에서 다룹니다.

### 학습 관련 질문

**Q: 이 교육을 완료하는 데 얼마나 걸리나요?**

모든 챕터를 실습과 함께 학습하면 약 **25-35시간** 정도 소요됩니다. 역할별 추천 경로를 따르면 핵심 내용을 **5-20시간** 내에 학습할 수 있습니다. 한번에 다 끝내려 하기보다 매일 1-2챕터씩 꾸준히 진행하는 것을 권장합니다.

**Q: 실습 환경은 어떻게 준비하나요?**

아래 "실습 준비" 섹션의 설치 가이드를 따르세요. macOS, Linux, Windows 모두 지원됩니다. 특별한 하드웨어 요구사항은 없으며, 일반적인 개발 환경이면 충분합니다.

**Q: 교육 내용이 업데이트되면 어떻게 알 수 있나요?**

이 교육 자료는 Git 저장소에서 관리되며, Claude Code와 AI 코딩 생태계의 변화에 맞춰 지속적으로 업데이트됩니다. 저장소를 watch하거나 주기적으로 pull하여 최신 내용을 확인하세요.

**능동적으로 실험하세요**

**자신만의 CLAUDE.md를 만드세요**

교육 중 배운 내용을 바탕으로 자신의 프로젝트에 적합한 CLAUDE.md를 작성해보세요. 이 과정에서 프로젝트 구조와 컨벤션을 더 깊이 이해하게 됩니다.

AI가 잘못된 코드를 생성하거나 의도를 잘못 파악한 경우, 그 원인을 분석하고 기록해두세요. "어떤 프롬프트가 왜 실패했는지"를 이해하면 프롬프트 작성 실력이 빠르게 향상됩니다.
